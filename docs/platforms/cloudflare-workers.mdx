The Cloudflare Workers platform with Durable Objects provides a robust environment for running RivetKit at the edge.

<MvpWarning />

## Deploy

<Steps>
  <StepStartFramework />

  <Step title="Install dependencies">
    Install the Cloudflare Workers platform package:
    
    <CodeGroup>
      ```sh npm
      npm install rivetkit/cloudflare-workers
      ```
      
      ```sh pnpm
      pnpm add rivetkit/cloudflare-workers
      ```
      
      ```sh yarn
      yarn add rivetkit/cloudflare-workers
      ```
      
      ```sh bun
      bun add rivetkit/cloudflare-workers
      ```
    </CodeGroup>
  </Step>

  <Step title="Create your server">
    Create a file `src/index.ts` to handle Cloudflare Workers integration:

    ```typescript src/index.ts
    import { createHandler } from "@rivetkit/cloudflare-workers";
    import { app } from "../workers/app";

    // Create handlers for Cloudflare Workers
    const { handler, WorkerHandler } = createHandler(app);

    // Export the handlers for Cloudflare
    export { handler as default, WorkerHandler };
    ```
  </Step>

  <Step title="Create KV namespace">
    Create a KV namespace for your workers:

    ```sh
    npx wrangler kv:namespace create WORKER_KV
    ```

    Take note of the KV namespace ID from the output, as you'll need it in the next step.
  </Step>

  <Step title="Configure Wrangler">
    Create a `wrangler.jsonc` configuration and add your KV namespace ID from the previous step:

    ```json wrangler.jsonc
    {
      "$schema": "node_modules/wrangler/config-schema.json",
      "name": "my-app",
      "main": "src/index.ts",
      "compatibility_date": "2025-04-04",
      "observability": {
        "enabled": true
      },
      "migrations": [
        {
          "tag": "v1",
          "new_classes": ["WorkerHandler"]
        }
      ],
      "durable_objects": {
        "bindings": [
          {
            "name": "WORKER_DO",
            "class_name": "WorkerHandler"
          }
        ]
      },
      "kv_namespaces": [
        {
          "binding": "WORKER_KV",
          "id": "your-namespace-id-here"  // Replace with your actual ID
        }
      ]
    }
    ```
  </Step>

  <Step title="Deploy to production">
    Deploy your project to Cloudflare Workers:

    ```sh
    npx wrangler deploy
    ```

    Your RivetKit application will be available at your Cloudflare Workers URL.
  </Step>

  <StepUpdateClient />
</Steps>

## Accessing Cloudflare Context And Bindings

You can access Cloudflare-specific features like the [DurableObjectState](https://developers.cloudflare.com/durable-objects/api/state/) and environment bindings from your worker:

```typescript
import { worker } from "rivetkit";

const myWorker = worker({
  // Load Cloudflare-specific variables
  createVars: (c, cloudflare) => ({
    ctx: cloudflare.ctx,
    env: cloudflare.env,
  }),
  actions: {
    foo: async (c) => {
      // Access DurableObjectState
      await c.vars.ctx.storage.get("foo");

      // Access bindings
      await c.vars.env.MY_BUCKET.put(key, "foo");
    },
  }
});
```
